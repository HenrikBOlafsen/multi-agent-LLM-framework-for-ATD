Goal
Break the dependency cycle between `click/_compat.py` and `click/_winconsole.py` by removing exactly one static edge, using the least disruptive refactoring technique that preserves behavior and public APIs.

Important (exact text:) "Please refactor to break this cycle, without increasing architectural technical debt elsewhere (e.g., no new cycles). My ATD metric treats ANY module reference as a dependency (dynamic/lazy/type-only all count). So making imports dynamic or lazy is NOT sufficient. I care about architecture (static coupling), not runtime import order."

Why this exists (short)
The two modules form a circular dependency: `_compat.py` imports `_get_windows_console_stream` from `_winconsole.py`, while `_winconsole.py` imports `_NonClosingTextIOWrapper` from `_compat.py`. This creates tight architectural coupling that makes future changes harder and increases risk of breaking changes.

Technique
To break the cycle, we will apply the **dependency inversion via a provider function/parameter** technique. Specifically:

1. **Move `_NonClosingTextIOWrapper` definition out of `_compat.py` into a new private helper module** (`click/_private_helpers.py`) that has no dependencies on either `_compat.py` or `_winconsole.py`. This ensures no new cycles are introduced.

2. **Modify `_winconsole.py` to stop importing `_NonClosingTextIOWrapper` directly** and instead accept it as a parameter in the relevant functions (`_get_text_stdin`, `_get_text_stdout`, `_get_text_stderr`).

3. **Provide `_NonClosingTextIOWrapper` from `_compat.py` when calling these functions**, effectively inverting the dependency so that the consumer (`_winconsole.py`) now provides the needed type/class rather than depending on it.

4. **Update the call sites in `_winconsole.py`** to pass `_NonClosingTextIOWrapper` explicitly to the functions that need it.

5. **Ensure backward compatibility by re-exporting `_NonClosingTextIOWrapper` from `_compat.py`** so existing external users continue to work unchanged.

Any remaining changes needed to make sure the rest of the codebase will work fine after the change:
- Remove `_NonClosingTextIOWrapper` from the global namespace of `_compat.py` (since it's now in `_private_helpers.py`)
- Add a new file `_private_helpers.py` containing only `_NonClosingTextIOWrapper`
- Update imports in `_winconsole.py` to remove direct import of `_NonClosingTextIOWrapper`
- Modify `_get_text_*` functions in `_winconsole.py` to accept `_NonClosingTextIOWrapper` as a parameter
- Pass `_NonClosingTextIOWrapper` from `_compat.py` when calling those functions
- Re-export `_NonClosingTextIOWrapper` in `_compat.py` to maintain public API

Scope & guardrails
