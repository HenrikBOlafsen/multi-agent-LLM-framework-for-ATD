Goal  
Break the architectural cycle between `click/_compat.py` and `click/_winconsole.py` without changing behavior or the public API.

Important  
Please refactor to remove this cycle **without** introducing new cycles elsewhere. 
My ATD metric treats **any** module reference as a dependency (dynamic/lazy/type-only all count). Making imports lazy is **not** sufficient. Functionality must be identical after the change.

Why this exists (short)  
- `_compat.py` (under Windows) imports `_get_windows_console_stream` from `._winconsole`.  
- `_winconsole.py` imports `_NonClosingTextIOWrapper` from `._compat`.  
This creates a bidirectional dependency (`_compat` ↔ `_winconsole`) in the static graph.

Technique (describe what to do, not exact code)  
1) **Extract `_NonClosingTextIOWrapper` into a new private, dependency-free module**, e.g. `click/_textio.py`.  
   - This module should only depend on stdlib (`io`, `typing`) and must **not** import from `click.*`.  
   - Move the class implementation verbatim; keep behavior and docstrings/comments intact.

2) **Update imports**  
   - In `_winconsole.py`, replace `from ._compat import _NonClosingTextIOWrapper` with `from ._textio import _NonClosingTextIOWrapper`.  
   - In `_compat.py`, remove the class definition and import it from `._textio` where needed (replace local references accordingly).  
   - Keep `_compat.py`’s existing Windows-only import of `._winconsole._get_windows_console_stream` as-is.  
   Result: `_winconsole` no longer imports from `_compat`, so the cycle disappears.

3) **Do not change public API**  
   - `_NonClosingTextIOWrapper` is a private symbol; moving it is fine. No re-exports needed.  
   - All existing functions such as `get_text_stdin/stdout/stderr`, `open_stream`, etc., must behave exactly the same.  
   - No signature changes, no behavioral tweaks.

Scope & guardrails  
- Keep the diff minimal: add `click/_textio.py`, modify `_compat.py` and `_winconsole.py`.  
- No new third-party dependencies.  
- Preserve typing, comments, and error handling.  
- Ensure `mypy` / type hints still pass where applicable.

Done when  
- `_winconsole.py` **does not** import anything from `._compat`.  
- `_compat.py` imports `_NonClosingTextIOWrapper` from `._textio` (and still conditionally imports `_get_windows_console_stream` from `._winconsole` on Windows).  
- The static dependency graph shows **no cycle** between `_compat.py` and `_winconsole.py`, and **no new cycles** were introduced.  
- All tests pass, and common usage still works (e.g., `click.echo`, stream wrapping on Windows, ANSI handling).
